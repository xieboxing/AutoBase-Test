## 📋 系统提示词模板

### 主提示词（放入项目根目录的 `.claude/instructions.md`）

```markdown
# AutoBase-Test 智能测试系统

## 每次测试启动时执行

1. 读取以下文件了解项目状态：
   - `docs/Slickorps测试文档.md` - APP测试要求和历史
   - `data/knowledge/` - 测试知识库（元素定位、失败模式、优化记录）
   - `data/reports/latest-summary.json` - 上次测试摘要

2. 检查是否有新的测试要求待执行

3. 执行测试前，检查知识库是否有：
   - 更优的元素定位方式
   - 需要跳过的不稳定元素
   - 推荐的等待时间
   - 已知的失败模式和解决方案

## 测试执行中

1. 每个操作失败时，先尝试知识库中的备选方案
2. 如果仍失败，触发AI自愈：截图+DOM分析→获取新定位→更新知识库
3. 记录所有操作的耗时和结果

## 测试完成后执行

1. 生成测试报告（HTML + JSON）
2. 分析本次测试数据：
   - 哪些操作耗时最长
   - 哪些操作经常失败
   - 发现了哪些新元素
   - 哪些页面有变化

3. 更新知识库：
   - 新的元素定位方式
   - 失败模式及解决方案
   - 性能优化建议

4. 更新 `docs/Slickorps测试文档.md`：
   - 记录测试结果
   - 更新测试进度
   - 添加发现的问题

5. 如果有改进建议，输出优化摘要

## 知识库结构

```
data/knowledge/
├── elements.json        # 元素定位映射（主定位+备选定位）
├── failure-patterns.json # 失败模式库
├── optimization.json     # 优化记录
├── page-structures.json  # 页面结构快照
└── test-history.db       # SQLite测试历史
```

## 永久保存原则

所有测试数据永久保存在 `data/` 目录：
- 截图：`data/screenshots/`
- 报告：`data/reports/`
- 知识库：`data/knowledge/`
- 日志：`data/logs/`
```

## 🔄 自动优化工作流

### Phase 1: 测试前准备

```
┌─────────────────────────────────────────────────────────┐
│  读取测试文档 → 检查知识库 → 载入历史数据 → 规划测试    │
└─────────────────────────────────────────────────────────┘
```

**Claude 执行脚本**:
```
请执行以下步骤开始测试：

1. 读取 `docs/Slickorps测试文档.md`，找到所有 [ ] 未完成的测试项
2. 读取 `data/knowledge/elements.json`，加载已知元素定位
3. 读取 `data/knowledge/optimization.json`，应用之前的优化建议
4. 按优先级排序测试项，开始执行
```

### Phase 2: 智能测试执行

```
┌─────────────────────────────────────────────────────────┐
│  执行测试 → 失败时自愈 → 记录数据 → 实时优化             │
└─────────────────────────────────────────────────────────┘
```

**自愈机制**:
```
当元素定位失败时：
1. 截图当前页面
2. 获取页面DOM结构
3. 发送给AI分析：已知元素"{元素描述}"定位失败，请分析截图和DOM，给出新的定位方式
4. 用新定位重试
5. 成功后更新 knowledge/elements.json
```

### Phase 3: 测试后分析与优化

```
┌─────────────────────────────────────────────────────────┐
│  分析结果 → 识别模式 → 生成优化建议 → 更新知识库        │
└─────────────────────────────────────────────────────────┘
```

**Claude 执行脚本**:
```
测试完成后，请执行以下分析和优化：

1. 分析本次测试数据，生成报告
2. 对比上次测试，识别：
   - 新增的失败项
   - 已修复的问题
   - 性能变化
3. 更新知识库：
   - 保存成功的元素定位
   - 记录失败模式和解决方案
   - 更新优化建议
4. 更新测试文档：
   - 标记完成的测试项
   - 记录发现的问题
   - 补充新的测试要求
```

## 🔧 实现脚本

### 创建知识库初始化脚本

在 `scripts/init-knowledge.ts`:

```typescript
import fs from 'node:fs/promises';
import path from 'node:path';

const KNOWLEDGE_DIR = './data/knowledge';

async function initKnowledgeBase() {
  await fs.mkdir(KNOWLEDGE_DIR, { recursive: true });

  // 元素定位映射
  await fs.writeFile(
    path.join(KNOWLEDGE_DIR, 'elements.json'),
    JSON.stringify({ version: '1.0', elements: {} }, null, 2)
  );

  // 失败模式库
  await fs.writeFile(
    path.join(KNOWLEDGE_DIR, 'failure-patterns.json'),
    JSON.stringify({ patterns: [] }, null, 2)
  );

  // 优化记录
  await fs.writeFile(
    path.join(KNOWLEDGE_DIR, 'optimization.json'),
    JSON.stringify({ optimizations: [], recommendations: [] }, null, 2)
  );

  // 页面结构快照
  await fs.writeFile(
    path.join(KNOWLEDGE_DIR, 'page-structures.json'),
    JSON.stringify({ pages: {} }, null, 2)
  );

  console.log('✅ 知识库初始化完成');
}

initKnowledgeBase();
```
